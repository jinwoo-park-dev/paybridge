NicePay cancellation actions

Provider payment ID stores the NicePay TID. Provider transaction or approval ID stores the approval code when available.

Full cancellation

Uses the stored NicePay TID to request a full cancellation and records the result in the shared reversal history.

Partial cancellation

Requests a partial cancellation against the stored TID. On the test MID, the remaining card limit may need to be released in the merchant portal.

Stripe refund actions

Provider payment ID stores the Stripe PaymentIntent ID. Provider transaction or approval ID stores the latest charge ID when Stripe returns it.

Full refund

Creates a full refund against the stored charge or PaymentIntent and records the result in the shared reversal history.

Partial refund

Creates a partial refund for an explicit amount and updates the remaining reversible balance after the refund succeeds.
